# Security Policy

We take the security of sorelvi/stream-reader seriously. We appreciate your efforts to responsibly disclose your findings, and will make every effort to acknowledge your contributions.

## Supported Versions

We only support the latest major version of the library.
## Reporting a Vulnerability

If you have discovered a security vulnerability in this project, please DO NOT open a public issue. Publicly disclosing a vulnerability can put the entire community at risk.

Instead, please report it via email:

### 📧 [security@sorelvi.io](mailto:security@sorelvi.io)

## What to include in your report

Type of issue (e.g., buffer overflow, memory leak, injection).

Full paths of source file(s) related to the manifestation of the bug.

Step-by-step instructions to reproduce the issue.

Proof-of-concept or exploit code (if possible).

Impact of the issue, including how an attacker might exploit it.

## Our Response Policy

We will acknowledge receipt of your report within 48 hours.

We will provide an estimated timeframe for addressing the vulnerability.

We will notify you when the fix is released.

Thank you for helping keep stream-reader and our users safe!
